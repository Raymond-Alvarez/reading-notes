
### **CSS Transforms**

1.  **What does a CSS transform allow the developer to do to an element?**

    A CSS transform allows a developer to **position, size, and alter elements** on both a two-dimensional and three-dimensional plane. Using the `transform` property, elements can be distorted, rotated, scaled, or moved (translated) along the horizontal (x), vertical (y), and depth (z) axes.

2.  **Provide an example of a transform and how you could see that being used on a website.**

    An example is the **rotate** transform, which allows an element to be turned between 0 and 360 degrees. On a website, this could be used for a **"Card Flip" effect**, where an element rotates when a user hovers over it to reveal more information. Another practical use is a **3D shadow**, where `transform` and `translate` are used to make a button appear to "grow out of the screen" when hovered, providing tactile feedback to the user.

### **CSS Transitions & Animations**

1.  **What does a CSS transition allow the developer to do to an element?**

    A CSS transition allows a developer to **alter the appearance and behavior of an element whenever a state change occurs**. These changes are typically triggered by user interactions such as hovering (`:hover`), focusing (`:focus`), or clicking (`:active`).

2.  **How does a CSS animation differ from a CSS transition?**
    The primary difference is the level of control and complexity:
    *   **Transitions** provide a simple change from **one state to another**.
    *   **Animations** allow for **multiple points of transition** defined by `@keyframes`. This gives developers more control to create complex sequences that can repeat indefinitely or change directions.
